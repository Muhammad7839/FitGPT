Problem & Purpose
  • Outfit decision fatigue — The average person spends 17 minutes a day choosing what to wear, often feeling dissatisfied
  • Disorganized wardrobes — No centralized system to track, visualize, or plan clothing across seasons
  • Generic recommendations — Existing apps ignore body type, real-time weather, and personal style history
  • No explainability — AI suggestions without reasoning fail to build user trust or help users learn
  • Multi-platform gap — Wardrobe tools rarely span web and mobile with a shared, consistent backend
FitGPT is an AI-powered digital wardrobe platform that delivers personalized, explainable outfit recommendations based on your wardrobe items, preferences, body type, live weather, and occasion — across web and Android.

Challenges We Solved
⚠ Problem
Deployment instability and database persistence issues on cloud hosting
✓ Solution
Configured PostgreSQL persistence on Render with environment-based DB switching (SQLite local ↔ PostgreSQL production)
⚠ Problem
Authentication and API communication inconsistencies across Android and web
✓ Solution
Unified JWT token handling and standardized REST API communication between both clients via a shared backend
⚠ Problem
Wardrobe changes not reflected across platforms — web and Android out of sync
✓ Solution
Single shared FastAPI + PostgreSQL backend as the source of truth for all clients and platforms
⚠ Problem
AI model hallucinating wardrobe items the user doesn't own
✓ Solution
Backend validates all AI-returned item IDs against the real wardrobe and falls back to a local scoring algorithm on failure
